\begin{abstract}

    As complex classifiers become increasingly prevalent in sensitive domains, the ability to understand and 
    trust the decisions of these black-box models is essential. 
    This poster provides a comprehensive overview of current explainable AI (XAI) methods designed 
    to interpret classifier outputs. 
    First, logic-based methods attempt to formalize the classifier's reasoning to offer formal 
    guarantees on the quality and rigor of the produced explanations. Second, Shapley value 
    methods evaluate feature importance by assigning a contribution score to each input based on 
    game theory principles. Finally, perturbation methods observe model behavior by generating 
    perturbed input samples. These methods provide local explanations either through 
    linear approximations or user-friendly IF-THEN rules. 

\end{abstract}

\section*{Logic-based Abductive Explanations}

We consider a classifier $\kappa : F \to Cl$ where $F$ is the feature space and $Cl$ the set of classes. An instance $x \in F$ is described by a set of literals $(f_i, v)$. A partial assignment $E \subseteq x$ is a subset of literals of $x$, and $E(y)$ means that $y$ agrees with $E$ on all features it defines.

To define an Abductive explanation (AXp), we first need to define a weak AXp : Let $x \in F$ and $\kappa(x) = c$.
A weak AXp of $\kappa(x)$ is a partial assignment $E \subseteq x$ such that:
$$\forall y \in F,\; E(y) \Rightarrow \kappa(y) = \kappa(x)$$
An AXp is a subset-minimal weak AXp. Intuitively, an AXp is the smallest set of
feature values that is \textit{sufficient} to guarantee the prediction, regardless
of the values of the remaining features.
There is multiple methods to Find AXp, we will describe three of them here that work with blackbox model.

\subsection*{CPI-Xp (Coverage-based Explanations)}

Standard AXp's assume \textbf{feature independence}. When dependencies exist between features (e.g.\ a pregnant person is necessarily a woman), AXp's produce \textbf{superfluous or redundant} explanations \cite{cooper_abductive_2024}. CPI-Xp solves this by incorporating \textbf{feature constraints} and selecting explanations with \textbf{maximal coverage}.

\vspace{0.3em}
\textbf{Definition.} Let $F[\mathcal{C}]$ be the set of feasible instances under constraints $\mathcal{C}$, and $cov(E) = \{x \in F[\mathcal{C}] \mid E(x)\}$ the coverage of a partial assignment $E$.
A \textbf{CPI-Xp} of $\kappa(x)$ is a partial assignment $E$ such that:
\begin{enumerate}
    \item $E(x)$
    \item $\forall y \in F[\mathcal{C}],\; E(y) \Rightarrow \kappa(y) = \kappa(x)$
    \item No valid explanation $E'$ strictly subsumes $E$ (i.e.\ $cov(E) \subsetneq cov(E')$)
\end{enumerate}

The preferred variant \textbf{pCPI-Xp} keeps one representative per coverage class and is the only notion satisfying all seven formal properties (success, non-triviality, irreducibility, coherence, consistency, independence, non-equivalence).

\vspace{0.5em}
\textbf{Key characteristics:}
\begin{itemize}
    \item \textbf{Complexity:} $\Pi_2^P$-complete (exact), but a \textbf{polynomial approximation} exists via dataset-based versions (d-CPI-Xp) in $O(m^2 n^2)$.
    
\end{itemize}

\subsection*{Efficient and Rigorous Model-Agnostic Explanations (sbAXp)}

Classic logic-based XAI requires access to the model's internal structure and suffers from scalability issues. This method solves both by working \textbf{sample-based}: it only queries the model as a black-box on a finite sample $S \subseteq F$, producing \textbf{sample-based abductive explanations} (sbAXp).

\vspace{0.3em}
\textbf{Principle.} The method exploits the \textbf{duality between AXp and CXp}: an AXp answers ``Why this prediction?'' while a Contrastive Explanation (CXp) answers ``Why not another?''. A CXp is a minimal subset of features that, if changed, suffices to alter the prediction. From all sbCXp's, the sbAXp's can be recovered via \textbf{minimal hitting sets}, and vice-versa.

\vspace{0.3em}
\textbf{Definition.} A \textbf{sbAXp} is a subset $X \subseteq F$ such that:
$$\forall x \in S,\; \Bigl(\bigwedge_{i \in X} (x_i = v_i)\Bigr) \Rightarrow \kappa(x) = \kappa(v)$$
A \textbf{sbCXp} is a minimal subset $X$ such that $\exists x \in S$ agreeing with $v$ on $F \setminus X$ but $\kappa(x) \neq \kappa(v)$.

\vspace{0.5em}
\textbf{Key characteristics:}
\begin{itemize}
    \item \textbf{Complexity:} \textbf{Polynomial} $O(mn)$ to find one sbAXp --- the most efficient among logic-based methods.
    \item \textbf{Smallest sbAXp:} Finding the \textit{smallest} sbAXp is intractable in general, but solvable via a \textbf{MaxSAT} encoding in comparable time.
    \item \textbf{vs.\ CPI-Xp:} Much cheaper ($O(mn)$ vs.\ $\Pi_2^P$-complete), but lacks the 7 formal properties and does not handle feature dependencies.
    \item \textbf{vs.\ Argument-based:} Simpler algorithm (no argumentation framework needed), but does not filter contradictory explanations.
\end{itemize}

\subsection*{Argumentation-based Explanations}

This method uses an \textbf{argumentation framework} to generate and refine sbAXp's. Instead of computing explanations directly, it builds \textbf{arguments} (plausible sbAXp's) and eliminates contradictions through \textbf{attacks}, producing more robust explanations.

\vspace{0.3em}
\textbf{Principle.} Each argument has a \textbf{support} (a set of feature conditions) and a \textbf{conclusion} (a predicted class). Two arguments \textbf{attack} each other when their supports are compatible but their conclusions differ. All attacks are resolved simultaneously via \textbf{stable extensions} (maximal conflict-free sets). A selection function (e.g.\ Max, Card, $Incl_i$, $Incl_c$) filters the extensions, and an \textbf{inference rule} selects the final explanations:
\begin{itemize}
    \item \textbf{Universal inference:} keeps only arguments present in \textit{all} selected extensions --- guarantees coherence.
    \item \textbf{Existential inference:} keeps arguments present in \textit{any} selected extension --- more explanations but may break coherence.
\end{itemize}

\vspace{0.5em}
\textbf{Key characteristics:}
\begin{itemize}
    \item \textbf{Complexity:} \textbf{Polynomial} --- comparable to sbAXp.
    \item \textbf{Reliability:} ++ --- highest among logic-based methods, thanks to contradiction elimination.
    \item \textbf{vs.\ sbAXp:} Same complexity but \textbf{filters out contradictory explanations}, producing more robust results.
    \item \textbf{vs.\ CPI-Xp:} Much cheaper and more robust (++), but does not provide the full 7 formal properties and does not handle feature constraints.
    \item \textbf{Trade-off:} Coherence is \textbf{only guaranteed with universal inference}; existential inference may produce contradictions.
\end{itemize}

\section{Shapley values based XAI}

\newcommand{\mF}{\mathcal{F}}
\newcommand{\mP}{\mathcal{P}}
\newcommand{\mD}{\mathcal{D}}
\newcommand{\mbF}{\mathbb{F}}
\newcommand{\mbE}{\mathbb{E}}

This family of XAI methods are based on the Shapley values concept from game theory 
that give an \textbf{importance score} to each feature in the prediction of a single instance. Shapley values 
provide a mathematically fair and unique method to attribute the payoff of a cooperative game to the 
players. \\

Let $\mF = \{1, \dots, n\}$ be the set of the numbered features (the players). The Shapley value of a feature $i$ is computed by going through all the coalitions 
not containing $i$ and looking at the \textbf{payoff difference} with and without the feature $i$ involved \cite{merrick_explanation_2020} :

\begin{align}
    \phi_i = \frac{1}{n} \sum_{\mS \in \mF\backslash\{i\}} \binom{n-1}{|\mS|}^{-1} & \bigl( \mbE[f(X) | X_{\mS\backslash\{i\}} = x_{\mS\backslash\{i\}}] \\
    & - \mbE[f(X) | X_{\mS}= x_{\mS}] \bigr)
\end{align}

From the efficiency axiom of Shapley values \cite{fryer_shapley_2021}, we get the following property that states that the sum of the importance is equal to the actual predicted value minus a reference contribution 
that is the mean of all the predictions for the distribution on the input $X$

\begin{align}
    \sum_{i \in \mF} \phi_i(v) &= \mbE[f(X) | X_{\mF} = x_{\mF}] - \mbE[f(X) | X_{\emptyset} = x_{\emptyset}] \\
    &= f(x) - \mbE[f(X)]
\end{align}

Figure \ref{fig:shap-titanic} vizualises computed importance scores for the prediction of a naive Bayes 
classifier on an instance of the Titanic dataset.

\begin{center}
    \includegraphics[width=0.8\linewidth]{shap_titanic.png}
    \captionof{figure}{Feature importance attributions computed with Shapley values for the decision of a naive Bayes 
    model on an instance of the Titanic dataset. From \cite{strumbelj_efficient_2010}}
    \label{fig:shap-titanic}
\end{center}

\subsection*{Approximation}

\textcite{strumbelj_efficient_2010} have shown that computing exact Shapley values become intractable when the number of features is large ($n \geq 100$). \textcite{merrick_explanation_2020} have come up with sampling methods to get an approximation in polynomial time:
\begin{itemize}
    \item sub-coalitions are sampled by taking all the features preceiding $i$ in random orderings $O_j$ of 
$\mF$: $\mS_j = Pre_i(O_j)$ ;
    \item the reference instances $X$ are sampled following the distribution of input instances.
\end{itemize}

\section*{Perturbation model-agnostic method}

Perturbation methods identify which input features most significantly influence a 
black-box model's decision process. These model-agnostic techniques compare the model's 
output for an original instance against its predictions for perturbed samples. By observing 
how these modifications affect the classification, we can quantify the importance of each 
feature without accessing the model's internal architecture.

\subsection*{LIME}

LIME (Local Interpretable Model-Agnostic Explainer) generates local surrogates to approximate a 
black-box model $f$ \cite{ribeiro_why_2016}. 

\textbf{Key Principle}: It minimizes a loss function $\mathcal{L}(f,g,\pi_x)$ to ensure local fidelity
while keeping the explanation model $g$ simple enough ($\Omega(g)$) for humans.
A most global explanation of the black-box model can be provided by SP-LIME which 
select a set of local and non-redundant LIME explanations. 

\textbf{Limitation}: The explanation depends heavily on the kernel width $\pi_x$ and sampling, leading 
to potential instability.

\subsection*{Anchors}

Anchors were developed to solve the lack of clear boundaries provided by LIME. 
To perform that, Anchors generates IF-THEN rules which 'anchor' a prediction. 
This set of short and disjoint rules are more comprehensible and then preferred by 
the users \cite{ribeiro_anchors_2018}.

\textbf{Formal Definition}: A rule $A$ is an Anchor for $x$ if $A(x)=1$ and it maintains high precision $\tau$ among its neighbors 
    $ \mathbb{E}_{\mathcal{D}(z \mid A)}[1_{f(x)=f(z)}] \geqslant \tau $, 
with $ \mathcal{D}(x \mid A)$ the distribution of neighbors of $x$ which satisfy the Anchor $A$. 

Computing Anchors is a combinatorial optimization problem. To handle the exponential search space, a 
\textbf{Bottom-Up approach} is used:
\begin{itemize}
    \item \textbf{Candidate Generation}: Rules are extended iteratively with additional predicates.
    \item \textbf{Selection via MAB}: A Multi-Armed Bandit (\textit{KL-LUCB}) algorithm identifies 
    the most precise candidates using minimal model calls \cite{ribeiro_anchors_2018}.
    \item \textbf{Beam Search}: To avoid purely greedy results, the algorithm maintains $B$ candidates, 
    finally outputting the one with the \textbf{highest coverage}: $cov(A) = \mathbb{E}_{\mathcal{D}(z)} [A(z)]$.
\end{itemize}
